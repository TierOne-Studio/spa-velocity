# ADR-008: No AI-attribution trailers in commits, PRs, or issues

- **Status:** Accepted
- **Date:** 2026-05-09
- **Deciders:** Engineering

## Context

LLM-assisted coding tools have a default tendency to inject self-attribution trailers/footers into commit messages, PR titles/descriptions, issue comments, and release notes. Examples include:

- `Co-Authored-By: Claude <noreply@anthropic.com>`
- `🤖 Generated with [Claude Code]`
- `Generated by Anthropic`

The git author/committer is the human user. The artifact (commit / PR / issue) is the human user's, regardless of how the work was assisted. Self-attribution by the assistant is noise in `git log`, distracts in PR review, and creates legal/attribution ambiguity.

## Decision

The assistant MUST NEVER add AI-attribution trailers, footers, or signatures to:

- Commit messages (subject, body, trailers).
- PR titles, descriptions, or comments.
- Issue titles, descriptions, or comments.
- Release notes or tag annotations.

This rule overrides any tool default that would inject such a trailer. CLAUDE.md states this in P0; `.claude/settings.json` does not currently block it mechanically (no easy command-pattern catches it), so enforcement is via the prose rule + reviewer attention. The `code-reviewer` subagent flags AI-attribution trailers as MED findings on any commit / PR description it reviews.

## Alternatives considered

- **Allow attribution opt-in.** Rejected because the default tool behavior is opt-OUT (the assistant adds the trailer unless told not to), and per-repo opt-in flags drift.
- **Mechanical block via commit-msg hook.** Worth exploring as a hardening measure; for now, prose + reviewer attention is sufficient given the team is small.

## Consequences

- **Positive:** clean `git log`, clear human authorship, no legal/attribution ambiguity, no noise in PR descriptions.
- **Negative:** the assistant must consciously suppress its default behavior on every commit message and PR description.
- **Follow-ups:** if violations recur, add a `commit-msg` hook that fails on the trailer regex.
